‚úÖ The Solution

You need to configure Render to look for the dist directory instead of the default build directory.

1 Update your package.json

Make sure your package.json has the correct scripts and dependencies:

{
  "name": "cdclients",
  "homepage": "https://jemila2.github.io/cdclients/",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  },
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "@emotion/react": "^11.14.0",
    "@emotion/styled": "^11.14.1",
    "@heroicons/react": "^2.2.0",
    "@mui/icons-material": "^7.3.1",
    "@mui/material": "^7.3.1",
    "axios": "^1.10.0",
    "react-router-dom": "^7.8.2",
    "react-toastify": "^11.0.5"
  },
  "devDependencies": {
    "@vitejs/plugin-react": "^5.0.1",
    "vite": "^7.1.3",
    "gh-pages": "^6.3.0"
  }
}

2 Configure Render.com settings

In your Render dashboard, update the build settings:

  • Build Command: npm run build
  • Publish Directory: dist

Note: If you're using a static site on Render, make sure to set the Publish Directory to dist in your service settings.

3 Add a render.yaml file (optional)

For more control, create a render.yaml file in your project root:

services:
  - type: web
    name: cdclients
    env: static
    buildCommand: npm run build
    staticPublishPath: ./dist
    envVars:
      - key: NODE_VERSION
        value: 18.18.0
